## 8. Code-review convergence (post-implementation adversarial review)

Four parallel reviewer subagents (architecture, security, correctness, spec-compliance) flagged findings that were triaged skeptically against the actual code. The confirmed real findings are fixed here; false positives were rejected.

- [x] 8.1 **C1-correctness (CRITICAL):** `LiveRecordingState::is_recording()` was backed by a stale `RECORDING_FLAG: AtomicBool` set only by the legacy `start_recording` command — never by `start_recording_with_devices_and_meeting` (the production path). The entire dispatch state guard was dead (`[Stop recording]` did nothing, `[Continue]` started duplicates). Fixed: `is_recording()` now reads `audio::recording_commands::current_phase() == RecordingPhase::Recording` (the authoritative source set via `compare_exchange` by every start path). Stale flag + all stores deleted. Adversarial test `live_recording_state_reflects_authoritative_phase` added (would have caught the bug). Design Decision 7.
- [x] 8.2 **C1-arch/spec-compliance (CRITICAL):** Detector-started recordings showed manual wording — `show_meeting_detected` / `recording_detected` / `RecordStartSource::Detector` were dead code. Fixed: `detectorStarted: boolean` threaded from `useAutoDetect.ts:151` → `handleRecordingStart` → `recordingService.startRecordingWithDevices` → `start_recording_with_devices_and_meeting(detector_started: Option<bool>)` Tauri command → `show_recording_started_notification(.., source: RecordStartSource)` branch. Design Decision 3 amended.
- [x] 8.3 **C2/I3 (IMPORTANT):** Stopped-toast body was `"Recording has been stopped and saved"` (static, no title) vs spec's `"Recording saved: <meeting title>"`. Fixed: `recording_stopped(meeting_name: Option<String>)` formats the body; title threaded from `stop_recording`'s `result.meeting_name` → `show_recording_stopped_notification` → `manager.show_recording_stopped`. Design Decision 8.
- [x] 8.4 **I2 (IMPORTANT):** `show_action_toast` mixed XML construction with WinRT I/O — not unit-testable. Fixed: extracted `build_action_toast_xml(title, body, actions) -> String` pure function; 5 adversarial tests added (XML injection in title/body/label/URI, ampersand-in-URI escaping, empty actions). Design Decision 9.
- [x] 8.5 **M3 (MINOR):** Fragment `#` check ran after query stripping, so `stop?x=1#frag` was silently accepted not rejected. Fixed: check moved before query strip. Adversarial test added. Design Decision 10.
