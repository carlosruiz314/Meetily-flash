- [x] 5.1 Do NOT add a hosted CI smoke workflow; delete the `.github/workflows/smoke.yml` drafted earlier in this change. The merge gate is a local git hook, not a hosted runner (design D8).
- [x] 5.2 Add `.githooks/pre-push` at the repo root: derives the change name from `fix/<change>`|`enhance/<change>`, runs `pnpm test` (Vitest) always, and runs `pnpm exec playwright test e2e/smoke/<change>.spec.ts` only if that file exists; no-ops to Vitest-only when the branch has no `fix/`|`enhance/` prefix or `frontend/` is absent; `SKIP_SMOKE=1 git push` bypasses for WIP/emergency pushes.
- [x] 5.3 Self-install the hook on `pnpm install` via a `prepare` script in `frontend/package.json` (`node scripts/install-smoke-hook.mjs`) that sets `git config core.hooksPath .githooks` and chmods the hook where the filesystem supports it; no-ops outside a git repo so Docker builds are unaffected. Zero new dependencies (Node is already required). VERIFIED LOCALLY: installer sets `core.hooksPath=.githooks`, the hook's exec bit is set, and branch derivation resolves `enhance/recording-basic` → `recording-basic.spec.ts` (would run), `enhance/cross-platform-automated-smoke-tests` → no self-spec (Vitest-only), and `main` → Vitest-only.
- [x] 5.4 (validation) Verified the hook end-to-end on a real push to origin (commit `63baae7`, branch `main`): the hook fired, ran the full Vitest suite (211/211 green, including the untracked `meeting-max-speakers.test.ts`), resolved `main` as no-prefix → Vitest-only gate, and the push proceeded on green.
